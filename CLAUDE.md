# CLAUDE.md — onepercentbetter.poker

## What This Site Is

`onepercentbetter.poker` is not a portfolio.
It is a direct product front door for one poker tool:

- `Exploit Better`

Current MVP:

- one tournament in
- three habits out
- one next-session adjustment

The site should make a poker player understand the pitch in seconds:

- this tool reviews one tournament
- this tool finds three repeated habits
- this tool gives one next-session adjustment
- this tool helps you become 1% better than yesterday

If a visitor has to read too much to understand the product, the page is wrong.

## Core Positioning

Primary promise:

- One tournament. Three habits. One adjustment.

Emotional outcome:

- "You are 1% better than yesterday."
- "You became a better poker player."

This is not a GTO trainer.
This is not a theory-heavy education site.
This is not a generic hand history viewer.

This is an exploit-focused analysis product built around:

- your own recurring patterns
- fish pattern recognition
- clearer exploit decisions
- measurable growth over time

This is the right MVP because it is small, clear, and useful.
It should not try to prove everything.
It only needs to give the user one honest review loop they would pay for.

## Product Thesis

`Exploit Better` exists because standard poker study tools often stop at numeric output.
They show solver answers, charts, or isolated hand feedback.
They do not clearly bridge:

- your actual repeated habits
- the field's repeated mistakes
- the exploit that should follow

The product should feel like:

- pattern recognition
- leak tagging
- fish labeling
- exploit suggestion
- small compounding improvement

Not:

- academic
- bloated
- overly technical
- solver worship

## Story To Keep In Mind

The founder story matters, but should be compressed into product truth.

Important context:

- built by a real tournament player
- shaped by real frustration with weak study workflows
- driven by the belief that fish patterns do not change much, so the player must adapt better
- built because there is a real path to value and revenue here

The site can carry some of that energy, but must not become a long autobiography.
Story should support the product, not dominate it.

## What To Emphasize

- one product, not a suite
- one outcome, not a roadmap dump
- one clear idea: exploit better
- visual proof of concept over paragraphs
- stronger product energy than startup pitch energy

Best message pillars:

- One tournament
- Three habits
- One adjustment
- +1% improvement

## What To De-Emphasize

- `Action Keeper` as a primary product on this site
- `.dev` as a strong cross-sell
- broad personal brand storytelling
- too many sections
- too many badges, tags, or labels
- technical architecture details
- long roadmap explanations
- forms or feedback capture that make the page feel cheap

If another idea is not helping `Exploit Better` convert, remove it.

## Visual Direction

The current visual benchmark is closer to a sportsbook/product launch page than a SaaS dashboard.

Use:

- dark charcoal / near-black base
- strong red accent system
- sharp contrast
- clean blocks
- bold hierarchy
- immediate product framing

The page should feel:

- direct
- confident
- fast to parse
- serious
- modern

The page should not feel:

- soft
- artsy
- noisy
- over-illustrated
- over-written

Important:

- product visuals should look like believable end-state product concepts
- avoid decorative background patterns that distract from the offer
- motion should be minimal and purposeful

## Copy Rules

- Keep copy short.
- Prefer punchy lines over explanations.
- Use poker language only when it increases clarity.
- Every section must earn its place.
- If a sentence sounds like brand strategy instead of product value, cut it.

Good:

- Find the leak.
- Exploit it.
- Get 1% better.

Bad:

- long philosophy blocks
- generic innovation language
- vague "workflow" language with no outcome

## Conversion Rules

This site should convert into backing, not browsing.

Primary CTA:

- Back the build

Backer structure currently maps to:

- `Open`
- `3-Bet`
- `All-In`

These should feel like clear support levels for shipping `Exploit Better` faster.

Do not clutter the site with too many CTAs.
One primary action is enough.

## Product Scope Guardrails

Do not let the marketing site become bigger than the product.
Do not let the product become bigger than the first useful version.

For `Exploit Better`, prefer:

- one-tournament review
- three-habit summary
- one recommended adjustment
- post-session analysis
- leak detection
- fish pattern tagging
- simple exploit guidance
- visible progress signal

Avoid prematurely expanding into:

- long-term dashboards before the first report is valuable
- multi-tournament memory before single-tournament insight is strong
- giant training platform features
- social/community features
- heavy account systems unless needed
- unnecessary AI complexity
- too many unrelated poker tools

## MVP Output Rules

Every first-version report should feel like a coach review, not a stat dump.

Required structure:

1. tournament context
2. three repeated habits
3. one table or field pattern worth noting
4. one next-session adjustment

Good output:

- specific
- behavioral
- plain language
- grounded in observed hands
- useful within one read

Bad output:

- solver-heavy
- too many charts
- too many recommendations
- vague motivational fluff
- unexplained poker jargon

## Relationship To Other 1% Better Properties

- `.dev` is the broader maker/build brand
- `.poker` is the focused poker product brand

The two can feel related, but `.poker` should stand on its own.
When in doubt, optimize `.poker` for poker players, not for hiring managers.

## Default Decision Filter

When changing this site, ask:

1. Does this make `Exploit Better` easier to understand in under 10 seconds?
2. Does this make the product feel more real?
3. Does this help a poker player want to back it?
4. Is this simpler than before?

If the answer is not clearly yes, do not add it.
